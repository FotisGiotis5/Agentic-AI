# Αναφορά Πειράματος 1: Αξιολόγηση Δικτυακής Συμπεριφοράς Gemini API

## 1.1 Περιγραφή Πειράματος 
Το παρόν πείραμα αξιολογεί τη δικτυακή συμπεριφορά του μοντέλου **Gemini 1.5 Flash** μέσω κλήσεων API. Η μελέτη εστιάζει στη σύγκριση της κίνησης κατά τη διάρκεια αδράνειας (Docile) έναντι της ενεργής κίνησης (Active), καθώς και στον υπολογισμό της καθαρής δικτυακής καθυστέρησης (Network Latency).

## 1.2 Ανάλυση Δικτυακής Κίνησης
Η καταγραφή των δικτυακών πακέτων ανέδειξε δύο διακριτές φάσεις επικοινωνίας:

* **Φάση Αδράνειας (Docile Traffic):**
  * **Διάρκεια:** `0.0s` έως `71.9s`.
  * **Συμπεριφορά:** Η δικτυακή κίνηση ήταν σχεδόν μηδενική. Καταγράφηκαν αποκλειστικά πακέτα συντήρησης της σύνδεσης SSH (`Keep-Alive`), επιβεβαιώνοντας ότι η εφαρμογή δεν παράγει background noise κατά τη φάση της αναμονής.

* **Ενεργή Φάση (Active Traffic):**
  * **Έναρξη:** Στο δευτερόλεπτο `71.92`.
  * **Συμπεριφορά:** Καταγράφηκε άμεση αύξηση της κίνησης, η οποία ξεκίνησε με ερωτήματα πρωτοκόλλου DNS για την εύρεση της διεύθυνσης `generativelanguage.googleapis.com` και συνεχίστηκε με κρυπτογραφημένη ροή δεδομένων μέσω της θύρας 443.

## 1.3 Υπολογισμός Χρόνου Απόκρισης (Network Latency)
Για τον ακριβή υπολογισμό του χρόνου απόκρισης, απομονώθηκε ο θόρυβος του SSH και μελετήθηκαν τα ακραία χρονικά ορόσημα της συνομιλίας:

* **Start Time:** 71.922731 (Πακέτο #464: Ερώτημα DNS)
* **End Time:** 72.006042 (Πακέτο #487: Τελευταίο πακέτο TLS Application Data)
* **Συνολικό Latency:** 0.083311 (Ο καθαρός χρόνος ολοκλήρωσης της δικτυακής συναλλαγής)

## Αναφορά Πειράματος 2: Αξιολόγηση Πολυπλοκότητας (Task Complexity)

### 2.1 Περιγραφή Πειράματο
Το παρόν πείραμα αξιολογεί τη δικτυακή συμπεριφορά κατά την ανάθεση μιας σύνθετης εργασίας (παραγωγή αλγορίθμου Collision Detection για Tetris). Στόχος είναι η μέτρηση του επιπλέον δικτυακού φορτίου που απαιτείται για τη διαχείριση πολύπλοκων αποκριμάτων.

### 2.2 Ανάλυση Δικτυακής Κίνησης (Traffic Analysis)
Η καταγραφή ανέδειξε μια δυναμική ροή πακέτων:
* **Συνολικά Πακέτα:** 1142 πακέτα.
* **Χρόνος Απόκρισης (API Latency):** 36.84 δευτερόλεπτα.
* **Δικτυακή Συμπεριφορά:** Παρατηρήθηκε έντονη δραστηριότητα κατά το ανέβασμα του prompt, ακολουθούμενη από μια "περίοδο σκέψης" (inference idle time) και μια κλιμακούμενη ροή πακέτων κατά τη λήψη του κώδικα.

## 2.3 Συμπέρασμα
Η αύξηση της πολυπλοκότητας του task (από απλές ερωτήσεις σε παραγωγή κώδικα) αυξάνει δραματικά το Network Latency (από 0.08s σε 36.84s). Η ανάλυση αποδεικνύει ότι η επικοινωνία με τον AI agent δεν εξαρτάται μόνο από το δίκτυο, αλλά από τη χωρητικότητα του μοντέλου να παράγει δομημένο περιεχόμενο, το οποίο προκαλεί συνεχή ροή δεδομένων στην αλυσίδα επικοινωνίας.
